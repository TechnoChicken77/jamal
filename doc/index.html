Instructions

Instructions are the base of Jamal, nothing is possible without them. Here you can see all of the built-in instructions and their usage.

  • OUT(string arg1)

    • Used to output a string in arg1 to a terminal.

    Example: OUT("Hello World\n")

  • VAR(string arg1, string arg2, none arg3)

    • Used to create a variable with type named in arg1 and name of arg2. Value of a varible is passed via arg3.

    Example: VAR("int", "i1", 10)

  • I_TS(int arg1)

    • Used to convert an int into string.

    Example: I_TS(48)

  • RMSTK(int arg1)

    • Removes a memmory stack with adress/index in arg1.

    Example: RMSTK(2)

  • ADDOF(string arg1)

    • Returns an address/index of a variable.

    Example: ADDOF("var1")

  • CALL(string arg1, *args*)

    • Allows to call a section. Retuns whatever was returned by the section. All arguments after arg1 wil be turned into section arguments.

    Example: CALL("section1", 46, "some-string")

  • PUSH(int arg1, int arg2, none arg3)

    • Allows to write contents of arg3 to stack at arg1. Data will be written at adress of arg2.

    Example: PUSH(0, 16, $some_variable)

  • TAKE(int arg1, int arg2, int arg3)

    • Allows to read arg3 bytes starting from arg2 in stack located at arg1

    Example: TAKE(0, 8, 8)

  • UBIND(string arg1)

    • Unbinds variable with name of arg1 from section variables, while keeping its stack.

    Example: UBIND("var1")

  • BIND(int arg1, string arg2, string arg3)

    • Binds stack at arg1 to a variable named as arg3 with type of arg2.

    Example: BIND(0, "string" "s")

  • APPND(int arg1, none arg2)

    • Appends arg2 to stack at arg1.

    Example: APPND(ADDOF("hello") + " World!\n")

    ALLOC(int arg1)

    Retuns an empty stack with size of arg1.

    Example: ALLOC(16)

  • SIZEOF(int arg1)

    • Retuns size of the stack at ag1.

    Example: SIZEOF(2)

  • STRIPE(int arg1, int arg2, int arg3)

    • Removes all bytes of a stack except arg3 bytes located at arg2.

    Example: STRIPE(0, 8, 8)

  • IN()

    • Instruction used for keybpard input.

    Example: VAR("string", "user_input", IN)

  • Note, that there are also __INTOP and __STROP instructions, however, due to their specific use case, no documentation for them will be provided. These instructions weren't meant to be used outside of their specific use case.

Imports

Imports are used to load code that is located outside of the main file.

To import code,

#import

should be used.

Example: #import jbase.jml

If the file path contains spaces, \s shoud be used at their place. Also, for now there is no way to check what was already imported to prevent duplicate imports, however this feature will be comming in future.
